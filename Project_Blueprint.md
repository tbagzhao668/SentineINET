# SentinelAI 项目开发蓝图

## 1. 项目愿景
打造一个轻量级、智能化的防火墙自动化运维 Web 平台，通过 AI 赋能安全日志分析与策略下发，实现“发现-分析-封堵-备份”的自动化闭环。

## 3. 技术栈选型
- **Frontend**: React 18 + Tailwind CSS + Lucide Icons (图标) + Axios。
- **Backend**: Python 3.10+ + FastAPI + Netmiko (网络驱动) + SQLAlchemy (数据库)。
- **AI**: 支持 OpenAI, Gemini, Claude 等主流模型 API 接入。
- **OS**: 兼容 Windows/Linux/macOS 部署。

## 5. 核心流程设计
1. **引导配置**: Web 弹出 Step-by-step 向导，配置 SSH 凭据、AI API Key、日志过滤词。
2. **日志循环**: 后端定时或通过 Syslog 接收日志，利用预设关键词（如 Deny/Drop）初筛。
3. **AI 分析**: 过滤后的日志发送至 AI，输出 JSON 格式风险报告（包含攻击 IP、风险等级、封堵建议）。
4. **决策下发**: 用户在 Web 端勾选需要封堵的 IP，系统执行：`备份配置 -> 生成命令 -> SSH 下发 -> 验证连通性`。
5. **回滚机制**: 每次变更生成版本号，支持一键还原上一个备份文件。

## 6. 安全与隐私
- SSH 密码与 API Key 使用 AES-256 加密存储。
- 敏感操作必须经过 Web 端二次确认。

## 7. 阶段小结：LLDP 拓扑生成与交互（本次迭代）

### 7.1 已完成能力
- **后端设备接入与命令执行**：支持对 `127.0.0.1:2000/2001/2002` 等端口的 Telnet 按需连接（建立连接→执行命令→断开连接），并优化了非授权 Telnet 的输出接收（避免只拿到提示符）。
- **LLDP 命令候选集与缓存**：优先使用已验证的 Skill/缓存命令；按品牌提供命令候选集，提升跨厂商成功率。
- **拓扑生成的稳定兜底**：当大模型不可用或网络异常时，仍可根据采集到的 LLDP 输出在本地解析并生成拓扑（避免“必须依赖 AI 才能出图”）。
- **拓扑图交互（前端）**：
  - 鼠标滚轮缩放（基于 SVG `viewBox`）。
  - 鼠标拖动空白处平移视图（地图式拖拽）。
  - 节点拖拽任意摆放；位置保存在浏览器 `localStorage`，刷新后仍保留。

### 7.2 使用说明（面向操作）
- **缩放**：在拓扑画布上滚动鼠标滚轮。
- **平移**：在拓扑空白处按住左键拖动。
- **摆放设备**：按住某个设备节点拖动到任意位置。

### 7.3 一键化与体验优化建议（可作为下一迭代 Backlog）
#### A. 一键按钮（UI/交互）
- **一键重置视图**：将 `viewBox` 复位到初始范围（适合“拖远/缩太小找不到图”）。
- **一键重置布局**：清空手动节点坐标（同时清除 `localStorage` 中的拓扑位置），回到自动布局结果。
- **一键导出/导入布局**：将节点坐标导出为 JSON（便于多人共享或迁移浏览器/电脑）。

#### B. 交互细节增强
- **拖拽模式防误触**：当用户拖拽节点时临时禁用画布平移；当点击空白处才进入平移（目前已通过事件阻止冒泡降低误触，但仍可进一步强化）。
- **键盘辅助**：
  - `Space + 拖动`：强制平移；
  - `Ctrl + 滚轮`：强制缩放；
  - `Esc`：取消当前拖拽。
- **对齐与美观**：加入“吸附网格 / 吸附对齐线”，提升手动摆放的一致性。

#### C. 持久化与协作（更像“平台”）
- **保存到后端**：将手动节点坐标写入后端存储（按用户/拓扑范围/时间版本化），实现跨浏览器、跨账号一致。
- **版本管理**：为布局提供“保存版本/回滚版本”，与配置备份理念一致。

#### D. 可观测性与运维
- **一键诊断**：在拓扑页显示“最近一次采集时间、成功/失败设备数、失败原因汇总”，并提供一键复制排障信息。
- **端口占用提示**：前端启动端口冲突时给出明确指引（例如自动切换端口或提示释放占用）。
